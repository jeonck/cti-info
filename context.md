# My Context

> 이 문서는 수집된 자료의 "행동 판정" 기준이다.
> 여기 없는 주제는 원칙적으로 "무관" 또는 "학습"으로 판정한다.

## 내 환경/현황

> 판정의 1차 기준. "즉시조치/백로그"는 여기 직접 해당할 때만.

- CTI(Cyber Threat Intelligence)를 온톨로지·지식그래프 형태로 구조화하는 개인 연구/파이프라인을 운영 중
- 목표: 위협 정보(TTP, IOC, 취약점, 공격자 그룹)를 표준 스키마(STIX 등)로 정규화해 상호운용성을 확보하고,
  이를 바탕으로 잠재적 공격 시나리오를 분석하는 것
- 특정 조직의 실시간 방어 체계(SOC)를 운영하지는 않음 — 따라서 "즉시 패치해야 할 내 인프라"는 없지만,
  "구조화·분석 관점에서 즉시 확인해야 할 자료"는 즉시조치로 판정
- 결정 가능 범위: 개인 연구 우선순위 조정, 데이터 수집·모델링 방향 결정

## 관심 분야 (학습 판정 기준)

> 아래 주제는 "즉시조치/백로그" 대상 인프라가 없어도 "학습"으로 판정한다.

### CVE / 취약점·익스플로잇
- 신규 CVE 공개, CVSS 심각도, 익스플로잇 코드(PoC/Exploit-DB) 공개 여부
- CISA KEV(Known Exploited Vulnerabilities) 신규 등재
- 벤더 보안 권고, 패치 우선순위 관련 논의

### 위협 인텔리전스 구조화 / 표준·상호운용성
- STIX/TAXII, OpenCTI, MISP 등 CTI 데이터 모델·표준 관련 논의
- 온톨로지, 지식그래프로 위협 정보를 표현하는 방법론·연구
- IOC(침해지표) 공유 포맷, 정규화 기법

### 공격 기법(TTP) / 위협 행위자 동향
- MITRE ATT&CK 매핑 사례, 신규 TTP 분석
- APT/랜섬웨어 그룹의 캠페인 분석 리포트
- 악성코드 분석 리포트 (구조·행위 패턴 중심)

### 명시적 제외 (관심 분야처럼 보여도 무관 처리)
- 특정 보안 제품·서비스의 단순 홍보/마케팅 글
- CTI와 무관한 일반 IT 뉴스, 채용 공고성 글
- 이미 널리 알려진 기초 개념을 재탕하는 입문용 글 (신규 정보 없음)

## 판정 시 우선순위
1. CISA KEV 신규 등재, 활발히 익스플로잇 중인 Critical CVE, PoC 공개 직후의 고위험 취약점 → 즉시조치 후보
2. 신규 CVE(익스플로잇 미확인), 패치는 필요하나 긴급하지 않은 보안 권고 → 백로그 후보
3. 즉시조치·백로그에는 해당하지 않으나 "관심 분야"(CTI 구조화, TTP, 위협행위자 동향)에 해당 → 학습 후보
4. 그 외 일반 뉴스, 명시적 제외 항목, 마케팅성 글 → 무관
